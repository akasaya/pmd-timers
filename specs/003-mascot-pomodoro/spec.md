# Feature Specification: マスコットキャラクター応援ポモドーロアプリ

**Feature Branch**: `003-mascot-pomodoro`
**Created**: 2026-03-12
**Status**: Draft
**Input**: User description: "可愛い女の子（SDキャラの初音ミクみたいな感じ）がデスクトップのタスクバーの上でタスクとかを邪魔しない程度に応援するポモドーロテクニックのアプリを作るのはどうだろうか"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - タスクバー上のSDキャラクターと共に集中作業を始める (Priority: P1)

ユーザーがアプリを起動すると、SDスタイルのかわいい女の子キャラクターがデスクトップのタスクバー上端に登場する。キャラクターはタスクバーからひょっこり顔を出すような自然な表示形式で常駐し、作業の開始と終了にあわせて短いアニメーションやメッセージで静かに応援する。

**Why this priority**: キャラクターの存在と基本的な応援インタラクションがこのアプリのコアアイデンティティ。これがなければ通常のポモドーロアプリと差別化できない。

**Independent Test**: アプリを起動し、タスクバー上にSDキャラクターが表示され、タイマー開始・終了時に応援アニメーションが再生されることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態で、**When** 初回表示される、**Then** SDキャラクターがタスクバーの上端に配置され、通常の待機アニメーション（息をしているような小さな動き等）を繰り返す
2. **Given** キャラクターが待機中の状態で、**When** ポモドーロタイマーを開始する、**Then** キャラクターが短い応援アニメーション（手を振る・ガッツポーズ等）を再生し、「頑張って！」のような短いメッセージを吹き出しで表示する
3. **Given** タイマーが動作中の状態で、**When** 作業セッションが終了する、**Then** キャラクターがお疲れ様のアニメーションを再生し、「お疲れ様！休憩しよう」のような短いメッセージを表示する
4. **Given** キャラクターが表示されている状態で、**When** 他のアプリを操作する、**Then** キャラクターは作業の邪魔をせず、クリックやフォーカスを奪わない

---

### User Story 2 - キャラクターのメッセージがモチベーションを維持してくれる (Priority: P1)

作業中・休憩中・セッション完了のタイミングに応じて、キャラクターが状況に合った短いメッセージ（励まし・ねぎらい・休憩の促し）を吹き出しで表示する。メッセージは数秒で自動的に消え、作業の邪魔にならない。

**Why this priority**: 「応援する」という核心機能。キャラクターが表示されるだけでなく、意味のある言葉でユーザーを動機づけることが価値の源泉。

**Independent Test**: ポモドーロセッションの各フェーズ（開始・終了・休憩終了）を経て、それぞれに適したメッセージが表示され自動消去されることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** タイマーが動作中の状態で、**When** セッション中間地点（半分の時間が経過）に達する、**Then** キャラクターが「あと半分！頑張れ〜」のような中間応援メッセージを5秒程度表示する
2. **Given** メッセージが表示されている状態で、**When** 5秒が経過する、**Then** メッセージ吹き出しが自然にフェードアウトし、キャラクターは待機アニメーションに戻る
3. **Given** 4セッション完了した状態で、**When** 長い休憩に入る、**Then** キャラクターが特別な「4セッション完了おめでとう！」的なリアクションを見せる
4. **Given** 複数のポモドーロを繰り返している状態で、**When** 同じシチュエーションが来る、**Then** メッセージがランダムにバリエーションを切り替えて繰り返しにならない（各状況に最低3種類以上）

---

### User Story 3 - タイマーをキャラクターから直接操作する (Priority: P2)

ユーザーはキャラクターをクリックすることで操作メニューを呼び出し、タイマーの開始・停止・設定へのアクセスができる。別途タイマーウィンドウを開かなくてもキャラクターだけで主要な操作が完結する。

**Why this priority**: キャラクターを単なる装飾ではなくインタラクティブなUIとすることで使いやすさが向上。ただし基本タイマー機能が先に必要。

**Independent Test**: キャラクターをクリックして操作パネルが表示され、タイマー開始・停止が行えることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** キャラクターが表示されている状態で、**When** キャラクターをクリックする、**Then** タイマー操作パネル（開始/停止/リセット/設定）がキャラクターの近くにポップアップ表示される
2. **Given** 操作パネルが表示されている状態で、**When** パネル外の任意の場所をクリックする、**Then** 操作パネルが閉じる
3. **Given** 操作パネルが表示されている状態で、**When** 残り時間が表示されている、**Then** 現在のフェーズ名と残り時間（分:秒）が視認できる

---

### User Story 4 - キャラクターの設定をカスタマイズする (Priority: P3)

ユーザーはキャラクターのサイズ、タスクバー上の配置位置（左・中央・右）、メッセージ表示のオン/オフを設定できる。長期間使用する際に自分の好みに調整できる。

**Why this priority**: 使い続けるためのカスタマイズ。コア機能の後に実装可能で、デフォルト値が適切であれば多くのユーザーは設定不要。

**Independent Test**: 設定画面でキャラクターサイズと位置を変更し、タスクバー上の表示が即座に反映されることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** 設定画面が開いている状態で、**When** キャラクターサイズを「小・中・大」から選択する、**Then** キャラクターの表示サイズが即座に変更される
2. **Given** 設定画面が開いている状態で、**When** タスクバー上の配置位置（左端・中央・右端）を選択する、**Then** キャラクターの表示位置がタスクバー上で移動する
3. **Given** 「メッセージ表示をオフ」に設定した状態で、**When** タイマーイベントが発生する、**Then** キャラクターはアニメーションのみでメッセージ吹き出しは表示されない
4. **Given** カスタム設定が保存されている状態で、**When** アプリを再起動する、**Then** 前回の設定が引き継がれる

---

### Edge Cases

- タスクバーが画面の横（左右）や上に配置されている場合のキャラクター表示位置
- タスクバーが自動非表示（オートハイド）設定の場合のキャラクター挙動
- 複数モニター環境でのキャラクター表示対象ディスプレイの決定方法
- ゲームやプレゼンテーションの全画面表示中にキャラクターをどう扱うか
- タスクバーを他のアプリが占有している場合（通知領域の混雑）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはSDスタイルのかわいい女の子キャラクターをデスクトップのタスクバー上端に表示しなければならない
- **FR-002**: キャラクターは待機中・応援中・ねぎらい中の3種以上の異なるアニメーション状態を持たなければならない
- **FR-003**: システムはポモドーロタイマーのフェーズ変化（開始・中間・終了・休憩開始・休憩終了）を検知し、対応するキャラクターアニメーションを再生しなければならない
- **FR-004**: システムは各フェーズイベントに対応した応援メッセージを吹き出し形式でキャラクターの近くに表示しなければならない
- **FR-005**: 各状況（作業開始・作業終了・休憩開始・休憩終了等）に対して最低3種類以上のメッセージバリエーションをランダムで表示しなければならない
- **FR-006**: メッセージ吹き出しは表示から5秒後に自動フェードアウトしなければならない
- **FR-007**: キャラクターはクリック操作に対応し、タイマー操作パネル（開始/停止/リセット/残り時間表示/設定アクセス）をポップアップ表示しなければならない
- **FR-008**: キャラクターの通常表示状態（待機アニメーション）はCPU使用率に影響を与えないよう、軽量な動作でなければならない
- **FR-009**: キャラクターは他のウィンドウのクリックやフォーカスを妨げてはならない（インタラクションの邪魔をしない）
- **FR-010**: ユーザーはキャラクターサイズ・タスクバー上の配置位置・メッセージ表示のオン/オフを設定できなければならない
- **FR-011**: すべてのキャラクター設定はアプリ再起動後も保持されなければならない
- **FR-012**: タスクバーが自動非表示（オートハイド）により隠れている間、キャラクターはタスクバーの高さ分だけ下に移動して画面下端に表示され続けなければならない。タスクバーが再表示されたら、キャラクターは元のタスクバー上端の位置に戻らなければならない

### Key Entities

- **キャラクター状態**: キャラクターの現在のアニメーション状態（待機/応援/ねぎらい/特別）と表示中のメッセージ
- **メッセージセット**: フェーズ別のメッセージ文字列の集合。各フェーズに最低3種類のバリエーションを含む
- **キャラクター設定**: サイズ（小/中/大）、タスクバー上の位置（左端/中央/右端）、メッセージ表示フラグの設定データ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: キャラクターの待機アニメーション動作時のCPU使用率増加が1%未満に収まる
- **SC-002**: ポモドーロイベント発生からキャラクターのリアクション開始までが1秒以内である
- **SC-003**: メッセージ吹き出しが表示から5秒±0.5秒以内に自動消去される
- **SC-004**: 10セッション連続実施時に同じメッセージが3回以上連続して表示されない（バリエーション機能の検証）
- **SC-005**: キャラクターが表示されている状態でも、背後のアプリのクリック操作の成功率が100%維持される
- **SC-006**: ユーザー設定の変更がアプリ再起動後も100%正確に復元される

## Assumptions

- キャラクターはプロジェクト内蔵の2Dアニメーション素材を使用する（外部キャラクター素材のカスタムインポートは初期バージョンでスコープ外）
- デフォルトのキャラクターは「かわいいSDスタイルの女の子」として固定し、キャラクター変更機能は初期バージョンでスコープ外
- タスクバーはWindowsのデフォルト下部配置を想定する（上部・左右配置は将来対応とし、初期バージョンは下部配置のみ対応）
- キャラクターのデフォルトサイズは「中」、デフォルト配置は「右端」（システムトレイ近く）とする
- アニメーションはスプライトシートまたは軽量な画像シーケンス形式を前提とし、動画ファイル再生は不要
- タスクバーが自動非表示（オートハイド）の場合: キャラクターはタスクバー高さ分だけ下にスライドして画面下端に表示を維持し、タスクバー再表示時に元の位置に戻る
- 本機能は001-pomodoro-desktopの基本タイマー機能と連携し、タイマーイベントを受け取って動作する
