# Feature Specification: 休憩後の作業タイマー自動スタート設定

**Feature Branch**: `009-auto-resume-work`
**Created**: 2026-03-16
**Status**: Draft
**Input**: User description: "休憩終わったら次の作業のタイマーが自動でスタートする設定ができるようにできませんか？"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 設定画面でオート再開を有効にする (Priority: P1)

ユーザーが設定画面を開き、「休憩終了後に作業タイマーを自動スタートする」オプションをオン/オフできる。
有効にすると、休憩フェーズが終わった瞬間に手動操作なしで次の作業タイマーが自動的に始まる。

**Why this priority**: この設定の有無が本フィーチャーのコアであり、すべての価値を提供する唯一の機能。

**Independent Test**: 設定ダイアログでトグルをオンにし、ポモドーロを1サイクル走らせて休憩タイマーが終わったとき、自動で作業タイマーが始まることを確認できる。

**Acceptance Scenarios**:

1. **Given** 設定が「自動スタートOFF」の状態で、**When** 短い休憩タイマーが終了する、**Then** タイマーはIDLE状態で停止し、ユーザーが手動で開始ボタンを押すまで待機する
2. **Given** 設定が「自動スタートON」の状態で、**When** 短い休憩タイマーが終了する、**Then** 次の作業タイマーが自動的に開始され、タイマーが作業フェーズのカウントダウンを始める
3. **Given** 設定が「自動スタートON」の状態で、**When** 長い休憩タイマーが終了する、**Then** 次の作業タイマーが自動的に開始される
4. **Given** 設定が「自動スタートON」の状態で、**When** アプリを再起動する、**Then** 設定が保持されており、再起動後も自動スタートが有効である

---

### User Story 2 - 設定変更の即時反映 (Priority: P2)

ユーザーが実行中のセッションがある状態で設定を変更したとき、次のフェーズ遷移から新しい設定が適用される。

**Why this priority**: アプリ再起動なしに設定変更が有効になることで使い勝手が向上するが、P1が完成すれば十分な価値を提供できる。

**Independent Test**: 作業中に設定ダイアログを開いてオート再開をオンにし、次の休憩終了時に自動スタートが機能することを確認できる。

**Acceptance Scenarios**:

1. **Given** 作業タイマー実行中で「自動スタートOFF」、**When** 設定ダイアログでオンに変更して保存する、**Then** 次の休憩終了時から自動スタートが有効になる

---

### Edge Cases

- 作業中にユーザーが手動でリセット/停止した場合、自動スタートは発火しない（フェーズ完了ではないため）
- 自動スタートONで作業タイマーが自動開始されても、ユーザーは通常どおり手動で一時停止・停止できる
- 設定変更を「キャンセル」した場合、自動スタートの状態は変わらない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 設定画面に「休憩終了後に作業タイマーを自動スタートする」トグル設定を追加しなければならない
- **FR-002**: このトグルのデフォルト値は「オフ（無効）」でなければならない
- **FR-003**: ユーザーがこの設定を変更して保存したとき、設定ファイルに永続化されなければならない
- **FR-004**: アプリ起動時に設定ファイルから自動スタート設定を読み込み、動作に反映しなければならない
- **FR-005**: 自動スタートが有効なとき、休憩フェーズ（短い休憩・長い休憩とも）が正常終了した直後に作業タイマーを自動的に開始しなければならない
- **FR-006**: 自動スタートが無効なとき、休憩フェーズ終了後はIDLE状態に戻り、ユーザーの手動操作を待たなければならない

### Key Entities

- **BehaviorSettings**: アプリの動作設定グループ。`auto_start_next_session` フラグを含む（既存）
- **SettingsDialog**: ユーザーが設定を閲覧・変更するダイアログUI（既存。本フィーチャーでトグル追加対象）

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 設定ダイアログに自動スタートのトグルが表示され、1回の操作でオン/オフを切り替えられる
- **SC-002**: 自動スタートをオンにした状態で休憩タイマーが終了したとき、即座に（遅延なく）作業タイマーが自動開始される
- **SC-003**: 設定を変更してアプリを再起動した後も、設定値が保持されている
- **SC-004**: 自動スタートの有無にかかわらず、既存のポモドーロサイクル（作業→休憩→作業の流れ）の動作が壊れない

## Assumptions

- バックエンドエンジン（`timer_engine.py`）にはすでに `auto_start_next_session` フラグを参照するロジックが実装済みであり、設定値が `True` のとき休憩終了後に自動で作業セッションを開始する
- `BehaviorSettings.auto_start_next_session` フィールドも `AppSettings` の永続化処理もすでに存在する
- 本フィーチャーのスコープは「設定ダイアログへのUI追加と、保存時に既存フラグへ値を反映する配線」のみ
- 自動スタート時に追加のカウントダウン（例: 3, 2, 1…）は不要とする（必要なら別フィーチャーで検討）
