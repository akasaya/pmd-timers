# Feature Specification: 音声ミュートトグル

**Feature Branch**: `011-mute-toggle`
**Created**: 2026-03-16
**Status**: Draft
**Input**: User description: "通知音とBGMをワンタッチでミュートできる機能"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ワンタッチで全音声をミュートする (Priority: P1)

会議中や集中が途切れたくない瞬間に、ユーザーがウィジェットまたはトレイメニューからワンタッチで通知音・BGMをまとめてミュートできる。再度タップすると元に戻る。

**Why this priority**: ミュート機能の本質はこの「手間ゼロで音を止める」体験にある。設定ダイアログを開かずに即座に操作できることが最大の価値。

**Independent Test**: ウィジェットにミュートボタンを追加し、クリックで通知音・BGMが止まり、再クリックで戻ることを確認できる。

**Acceptance Scenarios**:

1. **Given** 音声が有効な状態で、**When** ミュートボタンをクリックする、**Then** 通知音・BGMが即座に無効になり、ミュート中であることがUI上で視覚的にわかる
2. **Given** ミュート中に、**When** 再度ボタンをクリックする、**Then** ミュート前の音声設定が復元される
3. **Given** BGMが再生中にミュートした、**When** ミュートボタンをクリックする、**Then** BGMが即座に停止する
4. **Given** ミュート中に休憩フェーズが終わった、**When** 通知タイミングになる、**Then** 通知音は鳴らない

---

### User Story 2 - ミュート状態を再起動後も保持する (Priority: P2)

ミュートにしたままアプリを再起動しても、ミュート状態が維持される。

**Why this priority**: 再起動のたびにミュートし直す手間を省く。ただしP1が完成していれば単純な設定永続化で実現できる。

**Independent Test**: ミュートにして再起動し、UIがミュート表示のままで音が鳴らないことを確認できる。

**Acceptance Scenarios**:

1. **Given** ミュート状態でアプリを終了した、**When** 再起動する、**Then** ミュート状態が維持されておりUIもミュート表示になっている

---

### Edge Cases

- ミュート中に設定ダイアログで通知音やBGMの個別設定を変更しても、ミュート解除時に変更後の設定が反映される
- 通知音・BGMどちらも無効（設定でオフ）の場合でもミュートボタンは機能する（操作自体は妨げない）
- ミュート解除時、BGMは自動再開しない（次のフェーズ切り替えで再生が始まる）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ウィジェット本体の操作ボタン列（開始/一時停止ボタンとスキップボタンの間）にミュートトグルボタンを追加しなければならない
- **FR-002**: ミュートをオンにすると通知音・BGMの両方が無効化されなければならない
- **FR-003**: ミュートをオフにすると、ミュート前の通知音・BGMの有効/無効設定が復元されなければならない
- **FR-004**: ミュート状態は設定ファイルに保存され、再起動後も保持されなければならない
- **FR-005**: ミュート中はUIにミュート中であることが視覚的に示されなければならない
- **FR-006**: BGMが再生中にミュートをオンにした場合、BGMの再生が即座に停止しなければならない

### Key Entities

- **MuteSettings**: ミュートのオン/オフ状態を保持する設定グループ（`is_muted: bool = False`）
- **BehaviorSettings**（既存）: 個別の `sound_enabled`・`bgm_enabled` フラグを保有。ミュート時はこれらを一時的に上書きする形で動作する

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ミュートボタンのクリックから音声停止まで体感的に即座（1秒以内）に完了する
- **SC-002**: ミュートのオン/オフを切り替える操作が1回のクリックで完了する
- **SC-003**: ミュート解除後、以前の音声設定（通知音・BGMの有効/無効）が正確に復元される
- **SC-004**: ミュート状態でアプリを再起動しても設定が保持されている

## Assumptions

- ミュートは通知音・BGMを「まとめて無効化」する機能であり、個別に制御する機能は既存の設定ダイアログに任せる
- ミュートボタンはウィジェット本体の開始/一時停止ボタンとスキップボタンの間に配置する（ホバー時に表示されるボタン列に追加）
- ミュート解除後のBGMは自動再開しない（次のフェーズ遷移で自然に始まる）
- ミュート設定は `BehaviorSettings` または新設 `MuteSettings` に追加する
