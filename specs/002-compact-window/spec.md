# Feature Specification: コンパクトウィンドウサイズ

**Feature Branch**: `002-compact-window`
**Created**: 2026-03-12
**Status**: Draft
**Input**: User description: "ちょっとしたサイズのウィンドウで"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 作業の邪魔にならない小さなウィンドウでタイマーを確認する (Priority: P1)

ユーザーはポモドーロタイマーを画面の隅に小さく表示しながら、残りの作業スペースを最大限に活用したい。タイマーは必要な情報（残り時間・フェーズ）のみを表示し、作業領域を圧迫しない。

**Why this priority**: ウィンドウの小ささはこの機能の核心。画面を広く使いながらタイマーを確認できることが最大のユーザー価値。

**Independent Test**: アプリを起動し、ウィンドウが画面の一角に収まる小さなサイズで表示され、残り時間とフェーズが判読できることで独立してテスト可能。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態で、**When** 初回表示される、**Then** ウィンドウは画面の5%以下の面積に収まるコンパクトなサイズで表示される
2. **Given** コンパクトウィンドウが表示されている状態で、**When** ユーザーが残り時間を確認する、**Then** 残り時間（分:秒）と現在のフェーズ（作業中/休憩中）が視認できる
3. **Given** コンパクトウィンドウが表示されている状態で、**When** ウィンドウ上でマウスホバーする、**Then** タイマー操作ボタン（開始/一時停止/リセット）が表示される
4. **Given** コンパクトウィンドウが表示されている状態で、**When** ユーザーがウィンドウ端をドラッグする、**Then** ウィンドウサイズを最小〜標準の範囲でリサイズできる

---

### User Story 2 - ウィンドウの透明度を調整して視覚的な邪魔を減らす (Priority: P2)

ユーザーはタイマーウィンドウの不透明度を調整し、作業中は半透明にすることで背景コンテンツを透かして見えるようにしたい。確認が必要なときだけ視認できる状態にできる。

**Why this priority**: コンパクトウィンドウをさらに目立たなくする補完機能。画面の邪魔を最小化するためのUX向上策。

**Independent Test**: 設定またはウィンドウ操作で透明度を変更し、ウィンドウ越しに背景が透けて見えることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** コンパクトウィンドウが表示されている状態で、**When** 透明度スライダーを操作する、**Then** ウィンドウの不透明度が20%〜100%の範囲でリアルタイムに変化する
2. **Given** 透明度が設定されている状態で、**When** マウスカーソルがウィンドウ上に移動する、**Then** ウィンドウが100%不透明になり操作しやすくなる
3. **Given** カスタム透明度が設定されている状態で、**When** アプリを再起動する、**Then** 前回設定した透明度が引き継がれる

---

### User Story 3 - クリックスルーモードで完全に作業の邪魔にならない (Priority: P3)

ユーザーはタイマーウィンドウをクリックスルー（マウスクリックが下のウィンドウに貫通する）モードにし、タイマーを視覚的には表示しつつ操作には全く干渉しない状態にしたい。

**Why this priority**: 一定のパワーユーザーに需要がある機能。通常のコンパクトウィンドウで対応できない場面で有用。

**Independent Test**: クリックスルーモード有効時に、タイマーウィンドウをクリックしても下のアプリが反応することで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** クリックスルーモードが有効な状態で、**When** タイマーウィンドウ上の任意の場所をクリックする、**Then** クリックはタイマーを通過して背後のウィンドウに届く
2. **Given** クリックスルーモードが有効な状態で、**When** 指定のホットキーを押す、**Then** クリックスルーモードが解除され、通常のウィンドウ操作が可能になる

---

### Edge Cases

- ウィンドウを最小化した際、残り時間がタスクバーやシステムトレイのアイコンに表示されるか
- 高DPIディスプレイ（4K等）でのウィンドウサイズと文字の可読性が確保されるか
- ウィンドウサイズを最小まで縮小したとき、必要な情報（残り時間）が欠けずに表示されるか
- クリックスルーモード中にタイマーが終了した場合、どのようにユーザーに通知するか

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは初回起動時に、画面の5%以下の面積に収まるコンパクトサイズでウィンドウを表示しなければならない
- **FR-002**: コンパクトウィンドウは残り時間（分:秒形式）と現在のフェーズ名を常に表示しなければならない
- **FR-003**: ユーザーはウィンドウ端をドラッグすることで、規定の最小〜標準サイズの範囲でウィンドウをリサイズできなければならない
- **FR-004**: ウィンドウサイズおよび画面上の位置の設定はアプリ再起動後も保持されなければならない
- **FR-005**: システムはマウスホバー時にタイマー操作ボタンを表示し、ホバーが外れると非表示にするオートハイドUI機能を提供しなければならない
- **FR-006**: ユーザーはウィンドウの不透明度を20%〜100%の範囲で調整できなければならない
- **FR-007**: マウスカーソルがウィンドウ上にある場合、ウィンドウは自動的に100%不透明になって操作しやすい状態にならなければならない
- **FR-008**: 透明度の設定はアプリ再起動後も保持されなければならない
- **FR-009**: ユーザーはホットキーでクリックスルーモードのオン/オフを切り替えられなければならない
- **FR-010**: クリックスルーモード中にタイマーが終了した場合、クリックスルーを一時解除して通知を表示し、ユーザーが確認後に元の状態に戻らなければならない

### Key Entities

- **ウィンドウ設定**: ウィンドウサイズ（幅・高さ）、画面上の位置（X・Y座標）、透明度、クリックスルーモードの状態を保持する設定データ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: コンパクトウィンドウが1920×1080の画面で占める面積が5%（約104,000ピクセル²）以下である
- **SC-002**: 最小サイズ表示時でも残り時間の文字が十分なサイズで判読可能であり、通常の視力のユーザーが50cm程度の距離から確認できる
- **SC-003**: 透明度変更操作から視覚反映まで0.1秒以内にリアルタイム反映される
- **SC-004**: ウィンドウサイズ・位置・透明度の設定が再起動後も100%正確に復元される
- **SC-005**: ホバー時のUI表示・非表示が0.2秒以内に切り替わり、操作の邪魔にならない

## Assumptions

- コンパクトウィンドウの「標準サイズ」上限の目安は幅250×高さ100ピクセル相当、最小サイズは残り時間が判読できる範囲（幅120×高さ50ピクセル相当）とする
- デフォルトの透明度は100%（完全不透明）とし、ユーザーが必要に応じて下げる
- ウィンドウのデフォルト表示位置は画面右上隅とする
- 本機能は001-pomodoro-desktopの基本タイマー機能の上に構築される拡張機能として位置づける
- タスクトレイへの最小化はスコープ外（別フィーチャーとして扱う）
