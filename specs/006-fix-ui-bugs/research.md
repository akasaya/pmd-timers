# Research: UIバグ修正

## バグ1: 透明度設定が逆

### 根本原因

`settings_dialog.py` のスライダーラベルが「透明度」（transparency）になっているが、内部的には「不透明度」（opacity）を制御している。

- スライダー値100% → `window_opacity = 1.0` → `setWindowOpacity(1.0)` = 完全不透明
- スライダー値20% → `window_opacity = 0.2` → `setWindowOpacity(0.2)` = ほぼ透明

「透明度100%」と表示されていると、ユーザーは「100%透明（見えない）」と解釈するが、実際は「100%不透明（完全に見える）」になる。

### 決定: ラベルを「不透明度」に変更

- **採用**: UI上の表示を「透明度」→「不透明度（はっきり度）」に変更
- **理由**: コードのロジックは正しい（値が大きい=くっきり）。表示ラベルを実態に合わせるだけで修正完了
- **代替案**: 値を反転する（100%→0.0, 20%→0.8に変換）- 不要な複雑化になるため却下

## バグ2: ホバーボタンのガタつき

### 根本原因

`enterEvent` / `leaveEvent` はウィジェットと**子ウィジェット**の境界でも発火する。マウスがボタン上に乗った瞬間、親ウィジェットの `leaveEvent` が発火してボタンが消え始め、その後 `enterEvent` が再発火するループが起きる。

### 決定: デバウンスタイマー + underMouse チェック

- **採用**: `leaveEvent` 発火時に即座にフェードアウトするのではなく、短時間（150ms）待ってから `underMouse()` を確認し、まだマウスがウィジェット内にあればキャンセルする
- **理由**: 子ウィジェット間の移動時の誤検知を排除できる最もシンプルな方法
- **代替案**: `QEvent::HoverMove` を使うアプローチ - イベントフィルターが必要で複雑
- **代替案**: `setAttribute(Qt.WA_Hover)` + `event()` オーバーライド - 効果はあるが実装量が多い

## バグ3: 文字の視認性

### 根本原因

ウィジェットは `WA_TranslucentBackground` 属性でフレームレス透明背景。テキストは `color: #ffffff`（白）のみ設定。明るい背景では白文字が見えない。

現在の構成:
- フェーズラベル: `color: #aaaaaa`（グレー → 白背景に溶け込む）
- タイマー数字: `color: #ffffff`（白 → 白背景に溶け込む）
- カウントラベル: `color: #888888`（グレー → 白背景に溶け込む）

### 決定: 半透明の背景パネルを追加

- **採用**: ウィジェット本体（またはテキスト領域）に半透明の暗い背景を設定する（例: `background: rgba(0,0,0,150)`）
- **理由**: テキストシャドウはQtの標準スタイルシートではサポートが限定的。背景パネルが最もシンプルで確実
- **代替案**: テキスト縁取り（`QGraphicsDropShadowEffect`）- 実装可能だが文字サイズが大きいと不自然
- **代替案**: ダーク/ライト2テーマ切り替え - 過剰実装

## バグ4: Windows統計データが表示されない

### 根本原因

`PyQt6-Qt6-Charts` は Windows 用の wheel が PyPI に存在しない（`Could not find a version`）。

- `session_bar_chart.py` は try/except でフォールバックラベルを表示する → ダッシュボードウィンドウ自体は開く
- ただしフォールバックラベルは「PyQt6-Qt6-Charts が必要です」という技術的メッセージのみ
- 統計カード（完了数・作業時間）は表示されるはずだが、Windowsで初回起動の場合はセッションデータが0件なので空に見える

### 決定: グラフ代替をテキスト形式の棒グラフに強化

- **採用**: Charts非インストール時のフォールバックを「1行テキストのASCII/棒スタイル」に変更。各日付の完了数をラベルで表示
- **理由**: グラフがなくても統計データが視覚的に確認できる。Windows依存の問題を解決
- **代替案**: `pyqtgraph` などの代替チャートライブラリ - 新規依存関係の追加は回避
- **代替案**: matplotlib embedded - 重くなる、PyInstaller時の複雑化
