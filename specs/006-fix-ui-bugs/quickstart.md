# Quickstart: UIバグ修正 手動テストシナリオ

## シナリオ1: 透明度設定の確認

1. アプリを起動
2. 右クリック → 設定を開く
3. 不透明度スライダーが表示されることを確認（「透明度」ではなく「不透明度」）
4. スライダーを最大（100%）に設定 → OKを押す
5. **期待**: ウィジェットが完全にくっきり表示される
6. スライダーを最小（20%）に設定 → OKを押す
7. **期待**: ウィジェットがほぼ透明（うっすら）になる

## シナリオ2: ホバーボタンの安定性確認

1. アプリを起動
2. マウスをゆっくりウィジェット上に乗せる
3. **期待**: ボタンが滑らかにフェードインし、ガタつかない
4. ボタンが表示された状態でマウスを静止させる
5. **期待**: ボタンが消えずに5秒以上表示され続ける
6. ボタンをクリックする
7. **期待**: タイマーが開始/停止する
8. マウスをゆっくりウィジェット外に出す
9. **期待**: ボタンが滑らかにフェードアウト

## シナリオ3: 文字の視認性確認

1. 明るい壁紙（白や薄い色）の上にウィジェットを配置
2. **期待**: タイマー数字・フェーズラベルが読める
3. 透明度を50%程度に設定
4. **期待**: 半透明状態でも文字が判読できる

## シナリオ4: Windowsダッシュボード確認（Windows環境）

1. WindowsでPyQt6（Charts無し）でアプリを起動
2. Ctrl+D またはメニュー → ダッシュボードを開く
3. **期待**: ダッシュボードウィンドウが開く
4. グラフエリアを確認
5. **期待**: 「PyQt6-Qt6-Charts が必要です」という技術メッセージではなく、テキスト形式の日別統計が表示される（データが0件でも「データなし」等の代替メッセージが表示される）
