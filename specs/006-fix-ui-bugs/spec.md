# Feature Specification: UIバグ修正 - 透明度・ホバー・文字視認性・Windows統計

**Feature Branch**: `006-fix-ui-bugs`
**Created**: 2026-03-15
**Status**: Draft
**Input**: User description: "透明度の数字が大きいほど表示がはっきりするから逆では？ 当たり判定が微妙でガクガクするのでボタンが押しづらい。文字が背景の色に溶け込んで見づらい。windowsだと統計データ見られない。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 透明度設定の直感的な動作 (Priority: P1)

ユーザーが設定画面で透明度を「高い数値」に設定したとき、ウィジェットがはっきりくっきり表示される。現在は逆になっており、数値が大きいほど透明（薄く）なってしまっている。

**Why this priority**: 透明度の意味が逆になっているため、ユーザーの直感に反する操作体験になっている。修正コストが低く影響が大きい。

**Independent Test**: 設定画面で透明度を最大値に設定し、ウィジェットが完全に不透明（くっきり見える）になることを確認できる。

**Acceptance Scenarios**:

1. **Given** 透明度設定が最大値（100%）、**When** ウィジェットを表示する、**Then** ウィジェットが完全に不透明で明確に見える
2. **Given** 透明度設定が最小値（20%）、**When** ウィジェットを表示する、**Then** ウィジェットがほぼ透明（薄く）表示される
3. **Given** 透明度設定を変更する、**When** 数値を上げる、**Then** ウィジェットがより不透明（はっきり）になる

---

### User Story 2 - ホバーボタンの安定した操作 (Priority: P1)

マウスをウィジェット上に乗せたとき、操作ボタンが安定して表示され、ガクガクせずにクリックできる。現在はホバー判定の境界付近でボタンが消えたり現れたりするため、クリックが困難。

**Why this priority**: ボタンが押せないのは操作不能に等しく、ユーザー体験を大きく損なう。

**Independent Test**: ウィジェット上にマウスを乗せてボタンが表示された後、そのまま静止してボタンをクリックできることを確認できる。

**Acceptance Scenarios**:

1. **Given** ウィジェットが表示されている、**When** マウスをウィジェット内に移動する、**Then** ボタンが表示されてガタつかない
2. **Given** ボタンが表示されている、**When** ボタン上にマウスを移動してクリックする、**Then** ボタンがクリックできる（消えない）
3. **Given** マウスがウィジェット外に出る、**When** 境界付近でゆっくり移動する、**Then** ボタンがなめらかにフェードアウトする（ガタつかない）

---

### User Story 3 - 文字の視認性向上 (Priority: P2)

タイマーの数字やラベルが背景の色と混じって読みにくい状態を解消する。任意の背景色や半透明状態でも文字がはっきり読める。

**Why this priority**: 時間が読めないとタイマーとしての基本機能が失われる。

**Independent Test**: ウィジェットを任意の背景（明るい・暗い）の上に置いて、タイマー数字が明確に読めることを確認できる。

**Acceptance Scenarios**:

1. **Given** ウィジェットが明るい背景（白い壁紙）の上にある、**When** タイマーを見る、**Then** 数字がはっきり読める
2. **Given** ウィジェットが暗い背景の上にある、**When** タイマーを見る、**Then** 数字がはっきり読める
3. **Given** ウィジェットが半透明状態、**When** タイマーを見る、**Then** 背景に関わらず文字が判読できる

---

### User Story 4 - Windows環境での統計ダッシュボード表示 (Priority: P2)

Windows環境でも統計ダッシュボードが正常に開いて表示される。現在はWindowsで統計データが表示されない。

**Why this priority**: 統計機能はアプリの主要機能の一つ。Windowsが主要対象OSであるため、動作しないことは重大な問題。

**Independent Test**: Windows環境でアプリを起動し、右クリックメニューまたはCtrl+Dでダッシュボードが開いて統計データが表示されることを確認できる。

**Acceptance Scenarios**:

1. **Given** WindowsでアプリがPython/exeから起動している、**When** 右クリック→ダッシュボードを選択する、**Then** ダッシュボードウィンドウが開く
2. **Given** ダッシュボードが開いている、**When** 過去のセッションが存在する、**Then** 統計データ（完了数・作業時間など）が表示される
3. **Given** PyQt6-Qt6-ChartsがWindowsにインストールできない、**When** ダッシュボードを開く、**Then** グラフの代わりに代替表示（テキストサマリー等）が表示される

---

### Edge Cases

- 透明度を変更した直後にホバーした場合、透明度とホバー時の不透明化が正しく連動するか？
- 長時間使用後にホバー判定がズレることはないか？
- セッション記録が0件の場合でもダッシュボードが開けるか？
- PyQt6-Qt6-Charts非インストール環境でも代替UIが表示されるか？

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 透明度設定の数値が大きいほどウィジェットが不透明（はっきり見える）になること
- **FR-002**: 透明度の最大値（100%）でウィジェットが完全に不透明になること
- **FR-003**: ホバー検出領域とボタン表示領域が一致し、ボタン上でマウスが止まっても非表示にならないこと
- **FR-004**: ホバー表示/非表示の切り替えがなめらかで、短時間に繰り返し切り替わるガタつきが発生しないこと
- **FR-005**: タイマー数字・ラベルテキストに背景との視認性を確保する装飾（縁取り・影・背景パネル等）を適用すること
- **FR-006**: Windows環境でダッシュボードウィンドウが正常に開くこと
- **FR-007**: PyQt6-Qt6-ChartsがインストールされていないWindows環境でも、ダッシュボードに代替コンテンツが表示されること

### Key Entities

- **透明度設定**: ユーザーが設定する0〜100%の値。100%=完全不透明、20%=ほぼ透明
- **ホバー状態**: マウスがウィジェット上にある/ない状態。ヒステリシス処理を加えてガタつきを防ぐ
- **ダッシュボードウィンドウ**: 統計データを表示するサブウィンドウ。クロスプラットフォームで動作すること

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 透明度設定を最大値にしたとき、ウィジェットが完全に不透明で表示される
- **SC-002**: ウィジェット上でマウスを静止させたとき、ボタンが5秒以上安定して表示され続ける（ガタつきなし）
- **SC-003**: タイマー数字が、明るい・暗いどちらの背景でも目視で判読できる
- **SC-004**: Windows環境でダッシュボードを開く操作が成功率100%（クラッシュや無反応なし）
- **SC-005**: PyQt6-Qt6-Charts非インストール環境でもダッシュボードが開き、何らかのコンテンツが表示される

## Assumptions

- 透明度の「逆」問題は、設定値をそのままウィンドウ透明度APIに渡しているため（100%=透明として扱われている）と推測される
- ホバーのガタつきは、マウスがボタン上に乗った瞬間にウィジェット外と判定される境界問題と推測される
- Windows統計表示の問題は、PyQt6-Qt6-ChartsのWindows wheel非提供によりインポートエラーが発生し、ウィンドウが開かない可能性が高い
- 文字の視認性問題は、フレームレスウィンドウで背景が透けて見えることが原因と推測される
