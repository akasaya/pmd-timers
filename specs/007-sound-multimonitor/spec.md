# Feature Specification: 通知音改善とマルチモニター対応

**Feature Branch**: `007-sound-multimonitor`
**Created**: 2026-03-16
**Status**: Draft
**Input**: User description: "通知音がならないので、何かフリーの音とか取ってきて。また自分で音を設定できると良いかもね。但し長すぎる場合は５秒で無音に。後、メインディスプレイにしか移動できないので、他のディスプレイにも移動できるようにしてほしい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - デフォルト通知音の再生 (Priority: P1)

セッション（作業/休憩）が終了したとき、音が鳴って終わりを知らせる。現在は通知音が鳴らない状態になっているため、バンドルされたフリーの音声ファイルをデフォルトとして使用する。

**Why this priority**: 通知音はポモドーロタイマーの核心機能。音がなければタイマーが終わったことに気づけない。

**Independent Test**: セッション終了時に音が再生されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 通知音が有効になっている、**When** 作業セッションが終了する、**Then** 音が再生される
2. **Given** 通知音が有効になっている、**When** 休憩セッションが終了する、**Then** 音が再生される
3. **Given** 通知音が無効になっている、**When** セッションが終了する、**Then** 音は再生されない
4. **Given** 音声ファイルが5秒より長い、**When** 再生開始から5秒経過する、**Then** 音が自動的に停止する

---

### User Story 2 - カスタム通知音の設定 (Priority: P2)

ユーザーが好みの音声ファイルを通知音として設定できる。設定ダイアログから音声ファイルを選択し、プレビュー再生して確認してから保存できる。

**Why this priority**: デフォルト音があれば基本需要は満たせる。カスタム音はユーザー体験の向上。

**Independent Test**: 設定ダイアログで音声ファイルを選択・プレビュー・保存し、次のセッション終了時にそのファイルが再生されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 設定ダイアログを開く、**When** 通知音の「参照」ボタンを押す、**Then** ファイル選択ダイアログが開く
2. **Given** 音声ファイルを選択した、**When** 「プレビュー」ボタンを押す、**Then** 選択した音が再生される
3. **Given** カスタム音声ファイルを設定して保存した、**When** セッションが終了する、**Then** 設定した音が再生される
4. **Given** 選択した音声ファイルが5秒を超える、**When** プレビュー再生する、**Then** 5秒で自動停止し、ファイル名の横に「5秒でカットされます」と表示される
5. **Given** カスタム音声ファイルのパスが無効になった（ファイル削除等）、**When** セッションが終了する、**Then** デフォルト音にフォールバックして再生される

---

### User Story 3 - マルチモニター対応 (Priority: P2)

複数ディスプレイを使用している環境で、ウィジェットをドラッグして任意のディスプレイに移動できる。現在はメインディスプレイの範囲内にしか移動できない制限がある。

**Why this priority**: 複数モニター環境で使いたい場合に必須。ただし音の問題ほど緊急ではない。

**Independent Test**: 2台以上のディスプレイ環境でウィジェットをドラッグし、サブディスプレイに移動してそこに留まることを確認できる。

**Acceptance Scenarios**:

1. **Given** 複数ディスプレイ環境、**When** ウィジェットをメインディスプレイの端を越えてドラッグする、**Then** サブディスプレイにスムーズに移動できる
2. **Given** ウィジェットがサブディスプレイに配置されている、**When** アプリを再起動する、**Then** サブディスプレイの同じ位置にウィジェットが表示される
3. **Given** ウィジェットをディスプレイ範囲外（全ディスプレイの外側）にドラッグする、**Then** 最も近いディスプレイの端に留まる（画面外に消えない）

---

### Edge Cases

- 音声ファイルの形式が対応していない場合（例：動画ファイルを選んだ場合）はデフォルト音にフォールバックする
- 再生中にアプリが終了した場合、音声リソースは適切に解放される
- シングルディスプレイ環境ではマルチモニター機能は変化なし（既存動作を維持）
- 音声ファイルが壊れている・読み込めない場合はデフォルト音にフォールバックする
- ディスプレイ設定変更（モニター切断等）でウィジェットが表示されなくなった場合、再接続時またはアプリ再起動時にメインディスプレイに移動する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: アプリにフリー・商用利用可能なデフォルト通知音声ファイルをバンドルすること
- **FR-002**: セッション終了時にデフォルト通知音を再生すること
- **FR-003**: 再生中の音声が5秒を超えた場合、5秒で自動停止すること
- **FR-004**: 通知音の有効/無効を設定から切り替えられること（既存機能との整合）
- **FR-005**: 設定ダイアログからカスタム音声ファイルを選択・保存できること
- **FR-006**: カスタム音声ファイルのプレビュー再生ができること
- **FR-007**: カスタム音声ファイルが5秒を超える場合、設定画面で警告表示すること
- **FR-008**: カスタム音声ファイルが利用不能な場合、デフォルト音にフォールバックすること
- **FR-009**: ウィジェットのドラッグ移動範囲を接続されている全ディスプレイの仮想デスクトップ全体に拡張すること
- **FR-010**: 再起動後もウィジェットの配置ディスプレイと位置を復元すること
- **FR-011**: ウィジェットが全ディスプレイ範囲外になる場合は最寄りのディスプレイ端にクランプすること

### Key Entities

- **通知音設定**: デフォルト音またはカスタム音声ファイルパスの選択状態。有効/無効フラグと合わせて保存
- **音声ファイル**: バンドルされるデフォルトファイル（assets/sounds/に格納）、またはユーザー指定のファイルパス
- **仮想デスクトップ境界**: 接続されている全ディスプレイを包含する矩形領域。ウィジェット移動範囲の基準

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: セッション終了時に100%の確率で通知音が再生される（音有効時）
- **SC-002**: 5秒を超える音声は必ず5秒以内に停止する
- **SC-003**: 設定ダイアログで音声ファイルを選択してから30秒以内にカスタム音の設定が完了できる
- **SC-004**: 複数ディスプレイ環境で全てのディスプレイにウィジェットを移動できる
- **SC-005**: 再起動後にウィジェットの位置が正確に復元される（ピクセル単位）

## Assumptions

- デフォルト音声ファイルはCC0またはパブリックドメインのシンプルな効果音（ベル・チャイム系）を使用
- 対応音声形式はWAV/MP3/OGGを基本とする
- マルチモニター検出はOSの提供するAPIを通じて行う（追加ライブラリ不要）
- 音声再生はpyerまたはQtのマルチメディア機能を使用（plyer経由の現行実装を優先）
