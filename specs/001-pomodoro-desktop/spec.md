# Feature Specification: Windowsデスクトップ ポモドーロタイマーアプリ

**Feature Branch**: `001-pomodoro-desktop`
**Created**: 2026-03-12
**Status**: Draft
**Input**: User description: "windowsのデスクトップアプリで常に表示していられるポモドーロ・テクニックのアプリを作りたい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - タイマーを開始して作業セッションを管理する (Priority: P1)

ユーザーはアプリを起動し、ポモドーロタイマー（デフォルト25分）を開始して集中作業を行う。タイマーが終了すると通知を受け取り、休憩フェーズに移行できる。

**Why this priority**: ポモドーロタイマーの基本機能であり、これなしではアプリとして成立しない。最も重要なユーザー価値。

**Independent Test**: アプリを起動してタイマーを開始し、カウントダウンが機能し、終了時に通知が表示されることで完全にテスト可能。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態で、**When** 「開始」ボタンを押す、**Then** 25分のカウントダウンタイマーが即座に開始され、残り時間が画面上に表示される
2. **Given** タイマーが動作中の状態で、**When** 25分が経過する、**Then** 音声通知とデスクトップ通知が表示され、「作業セッション完了」をユーザーに知らせる
3. **Given** タイマーが動作中の状態で、**When** 「一時停止」ボタンを押す、**Then** タイマーが停止し、残り時間がその時点で保持される
4. **Given** タイマーが一時停止中の状態で、**When** 「再開」ボタンを押す、**Then** 停止した残り時間からカウントダウンが再開される

---

### User Story 2 - 常に最前面に表示してマルチタスク中もタイマーを確認する (Priority: P1)

ユーザーはブラウザや他のアプリを操作しながら、ポモドーロタイマーを常に画面上に表示しておきたい。他のウィンドウを操作してもタイマーが隠れない。

**Why this priority**: 「常に表示していられる」というユーザーの核心ニーズ。これがないと通常のタイマーアプリと差別化できない。

**Independent Test**: タイマーウィンドウを起動し、他のアプリを全画面表示にしてもタイマーが最前面に残り続けることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態で、**When** 他のウィンドウをクリックして前面に表示する、**Then** タイマーウィンドウは他のすべてのウィンドウの上に常に表示される
2. **Given** タイマーが最前面表示になっている状態で、**When** ユーザーが「最前面固定を解除」する操作を行う、**Then** タイマーウィンドウは通常のウィンドウとして動作する
3. **Given** アプリが起動している状態で、**When** タイマーウィンドウをドラッグする、**Then** 画面上の任意の位置にウィンドウを移動できる

---

### User Story 3 - 休憩フェーズを管理してポモドーロサイクルを完了する (Priority: P2)

作業セッション完了後、ユーザーは短い休憩（5分）または長い休憩（15分）を取り、4セッション完了後に長い休憩を促されながらサイクルを繰り返す。

**Why this priority**: ポモドーロ・テクニックの核心的な仕組みであり、正式なポモドーロサイクルを実現するために必要。

**Independent Test**: 作業セッションを終了させ、自動的に休憩タイマーに移行し、休憩終了後に次のセッションを促すことで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** 作業セッションが完了した状態で、**When** アプリが次フェーズを判断する、**Then** 1〜3回目のセッション後は5分の短い休憩が自動的に開始される
2. **Given** 4回目の作業セッションが完了した状態で、**When** アプリが次フェーズを判断する、**Then** 15分の長い休憩が自動的に開始される
3. **Given** 休憩タイマーが動作中の状態で、**When** 休憩が終了する、**Then** 通知が表示され、次の作業セッションの準備ができたことをユーザーに知らせる

---

### User Story 4 - タイマー設定をカスタマイズして自分のペースに合わせる (Priority: P3)

ユーザーは作業時間・休憩時間のデフォルト値を変更し、自分の集中スタイルに合わせたポモドーロサイクルを設定できる。

**Why this priority**: 個人差に対応する機能。デフォルト値が合わないユーザーにとって重要だが、コア機能の後に実装可能。

**Independent Test**: 設定画面で作業時間を変更し、タイマーがその設定値でカウントダウンすることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** 設定画面が開いている状態で、**When** 作業時間を1〜60分の任意の値に変更して保存する、**Then** 次のタイマー開始時からその設定値が適用される
2. **Given** カスタム設定が保存されている状態で、**When** アプリを再起動する、**Then** 前回保存したカスタム設定が引き継がれる

---

### Edge Cases

- タイマー動作中にPCがスリープ状態になった場合、復帰後に正確な残り時間が表示されるか
- タイマー動作中にWindowsのセッションがロックされた場合、タイマーはどう動作するか（一時停止 or 継続）
- 作業時間に0分または不正な値が入力された場合、適切なエラーメッセージが表示されるか
- 複数ディスプレイ環境でウィンドウが2番目のモニターに表示されている状態で、そのモニターが切断された場合の挙動
- OSの通知設定でアプリの通知がブロックされている場合のフォールバック表示

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは25分の作業セッション、5分の短い休憩、15分の長い休憩をカウントダウンタイマーとして提供しなければならない
- **FR-002**: システムは他のすべてのウィンドウより前面にタイマーウィンドウを常に表示する「最前面固定」機能を持たなければならない
- **FR-003**: ユーザーはタイマーウィンドウを画面上の任意の位置にドラッグして移動できなければならない
- **FR-004**: システムはタイマー終了時に音声通知とデスクトップ通知の両方でユーザーに知らせなければならない
- **FR-005**: システムは4回の作業セッションを1サイクルとして自動追跡し、長い休憩のタイミングを判断しなければならない
- **FR-006**: ユーザーはタイマーの開始・一時停止・再開・リセット操作を行えなければならない
- **FR-007**: システムは作業セッション・短い休憩・長い休憩の時間をユーザーが設定画面からカスタマイズできる機能を提供しなければならない
- **FR-008**: ユーザーのカスタム設定はアプリ終了後も保持されなければならない
- **FR-009**: システムは現在のフェーズ（作業中・短い休憩・長い休憩）と完了したポモドーロ数を画面上に表示しなければならない
- **FR-010**: ユーザーはタスクバーのアイコンからアプリを最小化・復元できなければならない

### Key Entities

- **ポモドーロセッション**: 作業セッションの1単位。フェーズ（作業中/短休憩/長休憩）、開始時刻、予定時間、完了フラグを持つ
- **タイマー設定**: ユーザーが設定できるパラメータ。作業時間、短い休憩時間、長い休憩時間、長い休憩前のセッション数を含む
- **サイクル進捗**: 現在のセッション番号（1〜4）とその日の完了ポモドーロ総数を追跡するデータ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーはアプリ起動から30秒以内にタイマーを開始できる
- **SC-002**: タイマーの残り時間表示誤差が1秒以内であり、正確なカウントダウンが行われる
- **SC-003**: 最前面固定機能が有効な状態で、他のアプリを操作してもタイマーが100%の確率で最前面に表示され続ける
- **SC-004**: タイマー終了通知がセッション終了から3秒以内にユーザーに届く
- **SC-005**: アプリの起動時間が5秒以内であり、すぐに使用可能な状態になる
- **SC-006**: ユーザー設定の変更がアプリ再起動後も100%正しく保持される

## Assumptions

- ターゲットユーザーはWindowsデスクトップ環境（Windows 10以降）で個人の集中作業管理を目的としてアプリを使用する
- デフォルトのポモドーロ設定（作業25分・短休憩5分・長休憩15分・4サイクル）は業界標準に基づく
- 通知音はシステムデフォルトの通知音またはアプリ内蔵の音を使用する（外部音声ファイルのカスタマイズはスコープ外）
- スリープ復帰時はタイマーを一時停止状態として扱い、ユーザーに再開を促す（バックグラウンドでカウントを継続しない）
- タスクトレイ（システムトレイ）常駐機能はオプションとして後フェーズで検討し、初期バージョンには含めない
- 複数ユーザーのデータ共有やクラウド同期はスコープ外
