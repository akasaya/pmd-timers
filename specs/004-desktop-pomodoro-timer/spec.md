# Feature Specification: デスクトップ常駐ポモドーロタイマー

**Feature Branch**: `004-desktop-pomodoro-timer`
**Created**: 2026-03-15
**Status**: Draft
**Input**: User description: "デスクトップに表示し続けるポモドーロテクニックのアプリを作りたい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - ポモドーロセッションを開始して集中作業を始める (Priority: P1)

ユーザーがアプリを起動すると、デスクトップ上に常時表示されるタイマーウィンドウが現れる。ユーザーはタイマーを開始し、25分間の集中作業セッションを行う。セッション終了時に通知が届き、5分の休憩を促される。

**Why this priority**: これがアプリの中核機能。ポモドーロテクニックの基本サイクル（作業25分→休憩5分）が動作しなければ他の機能に意味がない。

**Independent Test**: アプリを起動してタイマーを開始し、25分後に「作業終了」の通知が届き、続けて5分の休憩タイマーが動作することで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態で、**When** 「開始」ボタンを押す、**Then** 25分のカウントダウンが開始し、タイマーが見える状態で動作し続ける
2. **Given** タイマーが動作中の状態で、**When** 25分が経過する、**Then** 作業セッション終了の通知が表示され、休憩の開始を促すメッセージが表示される
3. **Given** 休憩タイマーが動作中の状態で、**When** 5分が経過する、**Then** 休憩終了の通知が表示され、次の作業セッションの開始を促す
4. **Given** タイマーが動作中の状態で、**When** 「一時停止」ボタンを押す、**Then** タイマーが停止し、再開ボタンが表示される
5. **Given** タイマーが一時停止中の状態で、**When** 「リセット」ボタンを押す、**Then** タイマーが初期状態（25:00）に戻る

---

### User Story 2 - 他の作業をしながらタイマーを確認する (Priority: P1)

ユーザーが他のアプリを使いながらでも、タイマーの残り時間をいつでも確認できる。タイマーウィンドウは常に画面上に表示され続け、他のウィンドウに隠れない。

**Why this priority**: 「デスクトップに表示し続ける」という要件の核心。最前面表示がなければ通常のタイマーアプリと変わらない。

**Independent Test**: タイマーを起動後、他のアプリ（ブラウザ、エディタ等）を全画面で開いても、タイマーウィンドウが常に前面に表示されることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** タイマーが動作中の状態で、**When** 他のアプリウィンドウを最大化する、**Then** タイマーウィンドウが他のウィンドウより前面に表示され続ける
2. **Given** タイマーが最前面表示されている状態で、**When** デスクトップ上の任意の場所にドラッグする、**Then** タイマーウィンドウが指定した位置に移動する
3. **Given** タイマーが表示されている状態で、**When** アプリを再起動する、**Then** 前回の位置にタイマーウィンドウが表示される

---

### User Story 3 - セッション数を記録して達成感を得る (Priority: P2)

ユーザーはその日に完了したポモドーロ数（セッション数）を確認できる。4セッション完了ごとに長い休憩（15分）が自動的に提案される。

**Why this priority**: ポモドーロテクニックの重要な要素であるセッション管理と長休憩を提供することで、疲労管理と達成感を得られる。コア機能の後に価値を追加する機能。

**Independent Test**: 4回の作業セッションを完了した後、長休憩（15分）の通知が届き、当日の完了セッション数が表示されることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態で、**When** 作業セッションを完了する、**Then** 完了セッション数のカウンターが1増加する
2. **Given** 3セッションが完了した状態で、**When** 4セッション目が完了する、**Then** 通常の5分休憩ではなく15分の長休憩が提案される
3. **Given** セッションカウンターが表示されている状態で、**When** 日付が変わる（または手動リセットする）、**Then** セッションカウンターが0にリセットされる

---

### User Story 4 - タイマーの設定を自分好みにカスタマイズする (Priority: P3)

ユーザーは作業時間・短休憩時間・長休憩時間をそれぞれカスタマイズできる。デフォルト値（25分/5分/15分）は変更しなくても使えるが、自分のペースに合わせた調整が可能。

**Why this priority**: コア機能が動いてから後で追加できる拡張機能。デフォルト値で多くのユーザーは満足できる。

**Independent Test**: 設定画面で作業時間を20分に変更してセッションを開始し、20分後に作業終了通知が届くことで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** 設定画面が開いている状態で、**When** 作業時間を1〜60分の範囲で入力する、**Then** 次回のセッション開始時からその時間が適用される
2. **Given** カスタム設定が保存されている状態で、**When** アプリを再起動する、**Then** カスタム設定が引き継がれる
3. **Given** カスタム設定がある状態で、**When** 「デフォルトに戻す」ボタンを押す、**Then** 25分/5分/15分の標準設定に戻る

---

### Edge Cases

- 作業中にコンピュータがスリープ状態になった場合、タイマーはどう動作するか？（スリープ中は一時停止、復帰後に継続 or 経過時間を反映）
- 作業時間を0分や極端に長い時間（例：99時間）に設定しようとした場合の入力バリデーション
- タイマー動作中にアプリを閉じた（または強制終了した）場合の状態保存

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはデスクトップ上に常時表示されるタイマーウィンドウを提供しなければならない（他のウィンドウより前面に表示）
- **FR-002**: システムはポモドーロサイクル（作業→短休憩→作業→…→長休憩）を自動管理しなければならない
- **FR-003**: システムはセッション終了時にデスクトップ通知またはサウンドでユーザーに知らせなければならない
- **FR-004**: ユーザーはタイマーの開始・一時停止・リセットを操作できなければならない
- **FR-005**: ユーザーはタイマーウィンドウを画面上の任意の位置に移動できなければならない
- **FR-006**: システムは当日の完了セッション数を表示し続けなければならない
- **FR-007**: システムは4セッション完了後に長休憩（デフォルト15分）を自動的に提案しなければならない
- **FR-008**: システムはタイマーウィンドウの位置設定を次回起動時まで保持しなければならない
- **FR-009**: ユーザーは作業時間・短休憩時間・長休憩時間をカスタマイズできなければならない
- **FR-010**: システムはカスタム設定をアプリ再起動後も保持しなければならない

### Key Entities

- **ポモドーロセッション**: 作業セッション1回分。開始時刻・終了時刻・種別（作業/短休憩/長休憩）・完了状態を持つ
- **タイマー設定**: 作業時間・短休憩時間・長休憩時間・長休憩までのセッション数・通知のオン/オフを含むユーザー設定
- **日次記録**: その日の完了セッション数・累計作業時間・長休憩の回数

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーがアプリを起動してから最初のポモドーロセッションを開始するまでの操作が10秒以内に完了できる
- **SC-002**: タイマーが動作中に他のアプリを操作しても、タイマーの表示が途切れることなく常に確認できる
- **SC-003**: セッション終了通知がセッション終了から3秒以内に表示される
- **SC-004**: タイマーのカウントダウンが実際の経過時間と30秒以上ずれない
- **SC-005**: カスタム設定の変更がアプリ再起動後に正しく引き継がれる（設定保持率100%）

## Assumptions

- 対象OSはWindowsを主とするが、将来的にmacOS対応も想定する
- デスクトップ環境での使用を前提とし、スマートフォン対応は対象外
- ポモドーロテクニックの標準設定（25分作業/5分休憩/4サイクルで長休憩）をデフォルトとする
- コンピュータスリープ中はタイマーを一時停止し、復帰時に継続する
- 通知音はシステム標準サウンドを使用する（カスタム音源は対象外）
- インターネット接続不要のスタンドアロンアプリとする
