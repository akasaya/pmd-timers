# Feature Specification: コンパクトタイマーウィジェットと統計ダッシュボード

**Feature Branch**: `005-compact-stats-dashboard`
**Created**: 2026-03-15
**Status**: Draft
**Input**: User description: "ウィンドウサイズは他のウィンドウを邪魔しない程度の大きさにしたい。後統計データを表示するDashBoardがほしい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 他の作業を邪魔しない小さなタイマーウィジェットを使う (Priority: P1)

ユーザーはポモドーロタイマーを作業画面の隅に小さく表示し、他のアプリウィンドウの操作エリアを最大限に確保したい。タイマーは常に視認できるが、クリックの邪魔にならず、他のウィンドウの上に被さっても作業に支障をきたさない。

**Why this priority**: 「他のウィンドウを邪魔しない」はユーザーの明示的な要求。タイマーが作業を邪魔するようでは毎日使い続けられない。

**Independent Test**: アプリを起動し、タイマーウィジェットを起動した状態でブラウザやエディタを使い、ウィジェットが作業の邪魔にならないことを確認できる。

**Acceptance Scenarios**:

1. **Given** アプリが起動している状態で、**When** 初回表示される、**Then** タイマーウィジェットは画面の5%以下の面積に収まるコンパクトサイズで表示される
2. **Given** コンパクトウィジェットが表示されている状態で、**When** ウィジェット外の任意の場所をクリックする、**Then** ウィジェットはクリックを奪わず、背後のアプリに操作が届く
3. **Given** コンパクトウィジェットが表示されている状態で、**When** ウィジェット上にマウスをホバーする、**Then** タイマー操作ボタン（開始/停止/スキップ）が表示される
4. **Given** コンパクトウィジェットが表示されている状態で、**When** ウィジェット外にマウスが移動する、**Then** 操作ボタンが非表示になりウィジェットが最小表示（残り時間のみ）に戻る
5. **Given** ウィジェットが表示されている状態で、**When** ウィジェットをドラッグする、**Then** 画面上の任意の位置に移動でき、次回起動時もその位置に表示される

---

### User Story 2 - 統計ダッシュボードで作業記録を振り返る (Priority: P1)

ユーザーは専用のダッシュボード画面を開き、当日および過去のポモドーロ記録（完了セッション数・累計作業時間・中断率など）を視覚的に確認する。日々の集中度の変化を把握し、モチベーション維持や作業改善に役立てる。

**Why this priority**: 統計ダッシュボードはユーザーの明示的な要求であり、ポモドーロアプリの長期的な利用価値を高める核心機能。数字で成果が見えることでモチベーションが上がる。

**Independent Test**: タイマーを数セッション動かした後にダッシュボードを開き、当日の完了セッション数・累計作業時間・中断回数が正確に表示されることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** ダッシュボードを開いた状態で、**When** 当日の記録を確認する、**Then** 当日の完了セッション数・累計作業時間・短休憩回数・長休憩回数が表示される
2. **Given** ダッシュボードを開いた状態で、**When** 過去7日間のデータを参照する、**Then** 日別の完了セッション数がグラフ形式で表示され、傾向が視覚的に把握できる
3. **Given** ダッシュボードを開いた状態で、**When** 特定の日付を選択する、**Then** その日のセッション詳細（各セッションの開始・終了時刻・完了/中断の状態）が確認できる
4. **Given** セッションが進行中の状態で、**When** ダッシュボードを開く、**Then** タイマーはバックグラウンドで継続し、ダッシュボードがタイマーを止めることはない
5. **Given** ダッシュボードが開いている状態で、**When** 作業セッションが完了する、**Then** ダッシュボードの当日統計がリアルタイムで更新される

---

### User Story 3 - 週次・月次の集中傾向を把握する (Priority: P2)

ユーザーはダッシュボードの期間フィルターを切り替えて、週単位・月単位での作業パターン（曜日別の集中しやすい時間帯、最も生産的だった日など）を確認できる。

**Why this priority**: 7日・30日スパンで傾向を見ることで、単日では気づけないパターンが発見できる。コア機能の後に価値を追加する機能。

**Independent Test**: ダッシュボードの期間フィルターを「今週」「今月」に切り替えると、それぞれの期間に対応したグラフと集計値が表示されることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** ダッシュボードが開いている状態で、**When** 期間フィルターを「今週」に切り替える、**Then** 過去7日間の日別セッション数グラフと週合計値が表示される
2. **Given** ダッシュボードが開いている状態で、**When** 期間フィルターを「今月」に切り替える、**Then** 今月の週別・日別データが表示される
3. **Given** 過去データが存在する状態で、**When** ダッシュボードを開く、**Then** 最もセッション数が多かった日と、連続達成日数（ストリーク）が強調表示される

---

### User Story 4 - ウィジェットの透明度を調整して視覚的な邪魔を最小化する (Priority: P3)

ユーザーはタイマーウィジェットの透明度を調整して作業中の視覚的な邪魔を最小化できる。

**Why this priority**: コアのタイマー機能とダッシュボードが動いた後に追加できるカスタマイズ機能。

**Independent Test**: 設定から透明度を50%に変更するとウィジェットが半透明になり、次回起動時も引き継がれることで独立テスト可能。

**Acceptance Scenarios**:

1. **Given** ウィジェット設定が開いている状態で、**When** 透明度を20〜100%の範囲で設定する、**Then** ウィジェットの不透明度がリアルタイムで変化する
2. **Given** 透明度が設定されている状態で、**When** マウスカーソルがウィジェット上に移動する、**Then** ウィジェットが100%不透明に戻り操作しやすい状態になる
3. **Given** カスタム設定が保存されている状態で、**When** アプリを再起動する、**Then** 前回の透明度設定が引き継がれる

---

### Edge Cases

- データが0件の状態（初回起動直後）でダッシュボードを開いた場合の空状態表示
- 日付をまたいでタイマーが動作している場合、どちらの日付のセッションとして記録されるか
- 高DPIディスプレイ（4K等）でコンパクトウィジェットの文字が小さすぎて読めない場合
- ダッシュボードの期間に対応するデータが部分的にしか存在しない場合（使い始めの週など）

## Requirements *(mandatory)*

### Functional Requirements

**コンパクトウィジェット:**
- **FR-001**: タイマーウィジェットは画面の5%以下の面積（1920×1080基準）に収まらなければならない
- **FR-002**: ウィジェットは残り時間（分:秒）と現在フェーズ（作業中/休憩中）を常時表示しなければならない
- **FR-003**: ウィジェットはマウスホバー時のみ操作ボタンを表示し、それ以外は最小表示にならなければならない
- **FR-004**: ウィジェットの位置はアプリ再起動後も保持されなければならない
- **FR-005**: ウィジェットは他のウィンドウの操作（クリック・入力）を妨げてはならない

**統計ダッシュボード:**
- **FR-006**: システムは当日の完了セッション数・累計作業時間・中断回数・休憩回数を表示する専用ダッシュボード画面を提供しなければならない
- **FR-007**: ダッシュボードは過去のデータを日別・週別・月別に切り替えて表示できなければならない
- **FR-008**: ダッシュボードは日別セッション数をグラフ形式で視覚表示しなければならない
- **FR-009**: ダッシュボードはセッション継続中でも開け、タイマーを止めてはならない
- **FR-010**: ダッシュボードはセッション完了時にリアルタイムで統計を更新しなければならない
- **FR-011**: ダッシュボードは連続達成日数（ストリーク）を表示しなければならない
- **FR-012**: 過去90日分のセッションデータが保存・参照できなければならない

**共通:**
- **FR-013**: ウィジェットの透明度は20〜100%の範囲でユーザーが調整でき、設定が保持されなければならない

### Key Entities

- **ウィジェット設定**: 位置（X/Y座標）・透明度・サイズ（幅/高さ）・ホバー表示オン/オフ
- **日次統計**: 対象日・完了作業セッション数・中断セッション数・累計作業時間・累計休憩時間・長休憩回数
- **ストリーク**: 連続して目標セッション数（デフォルト1件以上/日）を達成した日数

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: コンパクトウィジェットが1920×1080画面で占める面積が5%（約104,000px²）以下である
- **SC-002**: ウィジェット表示中に隣接するウィンドウへのクリックが妨げられない
- **SC-003**: ダッシュボードをクリックで開いてから統計データが表示されるまで2秒以内
- **SC-004**: セッション完了からダッシュボードの当日統計が更新されるまで3秒以内
- **SC-005**: 過去90日分のデータが参照でき、グラフが正確な数値を反映している
- **SC-006**: ウィジェット設定（位置・透明度）がアプリ再起動後100%正確に復元される

## Assumptions

- 002-compact-window で定義されたコンパクトウィンドウの要件（透明度・クリックスルー）と本仕様は統合対象であり、実装時に重複する部分は一本化する
- ダッシュボードは独立したウィンドウ（モーダルまたは別画面）として開く。常時最前面表示は不要
- デフォルトの目標セッション数は「1日1件以上」とし、ストリーク計算に使用する
- ダッシュボードのグラフはシンプルな棒グラフを基本とし、過度に複雑な可視化はスコープ外
- データのエクスポート（CSV等）はスコープ外
- ウィジェットのデフォルト表示位置は画面右上隅（16pxオフセット）
- 統計データの保持期間は90日とし、それ以前のデータは自動削除される
