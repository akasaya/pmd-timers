## 2. グラフ描画ライブラリ

### 決定: PyQt6 の `QChart` / `QtCharts`（PyQt6-Qt6-Charts）

**根拠:**
- PyQt6 に同梱される `QtCharts` モジュールで棒グラフが描画可能
- 追加の外部依存なし（`PyQt6-Qt6-Charts` は PyQt6 の一部）
- `QBarSeries` + `QBarSet` で日別セッション数の棒グラフを実装できる
- アニメーション付き描画でポリッシュな見た目を実現
- セッション数程度の小データ量では性能上の問題はない

**比較検討:**

| ライブラリ | PyQt6との統合 | 棒グラフ | 追加依存 | ライセンス |
|-----------|-------------|---------|---------|-----------|
| QtCharts (PyQt6-Qt6-Charts) | ✅ ネイティブ | ✅ 対応 | なし | GPL/商用 |
| matplotlib + PyQt6 | ⚠️ FigureCanvas経由 | ✅ 対応 | matplotlib追加 | BSD |
| pyqtgraph | ✅ Qt統合 | ✅ 対応 | pyqtgraph追加 | MIT |

**却下した代替案:**
- **matplotlib**: FigureCanvas の埋め込みが必要で、Qt のネイティブウィジェットとのスタイル統一が難しい
- **pyqtgraph**: 主にリアルタイムデータ向けで静的な統計グラフには過剰

## 3. コンパクトウィジェットのホバーUI実装パターン

### 決定: `QWidget.enterEvent` / `leaveEvent` + `QPropertyAnimation`

**実装方針:**
- `enterEvent`: マウスがウィジェットに入ったら操作ボタン（QToolButton群）を `QPropertyAnimation` でフェードイン表示
- `leaveEvent`: マウスが離れたら操作ボタンをフェードアウトして非表示
- アニメーション時間: 150ms（操作の邪魔にならない速度）

**ホバー領域のポイント:**
- ウィジェット全体でホバーを検出（残り時間表示部分も含む）
- 子ウィジェット（ボタン）が表示された際のマウス離脱イベントを `setMouseTracking(True)` と親ウィジェット監視で正しく処理する

## 4. 統計集計パターン

### 決定: JSON ファイルから in-memory 集計

**根拠:**
- 90日 × 最大50セッション/日 = 最大4,500レコード
- このデータ量では SQLite や ORM は過剰。JSON + Python の dict 操作で十分
- 日付ファイル（`YYYY-MM-DD.json`）を必要な範囲だけ読み込んで集計

**集計ロジック:**

| 期間フィルター | 読み込むファイル数 | 集計方法 |
|-------------|----------------|---------|
| 今日 | 1件 | 当日ファイルから直接取得 |
| 今週（7日） | 最大7件 | 7ファイルを読み込んで日別集計 |
| 今月（30日） | 最大30件 | 30ファイルを読み込んで日別集計 |

**ストリーク計算:**
- 当日から過去に向かって連続で「完了セッション数 ≥ 1」の日を数える
- ファイルが存在しない日 = セッションなし = ストリーク途切れ

## 5. ダッシュボードウィンドウの表示方式

### 決定: モーダルレスな独立ウィンドウ（`QDialog` ではなく `QWidget`）

**根拠:**
- `QDialog` はモーダルにすると背後のタイマーウィジェットの操作を妨げる
- `QWidget` をトップレベルウィンドウとして使うことで、タイマーが動き続けながらダッシュボードを表示できる
- タイマーセッション完了時のシグナルをダッシュボードにも接続し、リアルタイム更新を実現
